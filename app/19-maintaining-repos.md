# Maintaining Repositories

---

In the professional *MERCWAR* ecosystem, maintaining repositories is a **strategic discipline**. Each repository is a node in the constellation, and its health directly impacts the stability of the entire network. Proper maintenance ensures discoverability, integrity, and readiness for integration.

---

### ⚙️ Core Maintenance Principles

| Principle | Focus | Function |
| --- | --- | --- |
| **Consistent Naming** | Identity | Use canonical names to prevent confusion and ensure cross‑repo recognition. |
| **Metadata Integrity** | Schema | Preserve AVIS headers, filename comments, and AIFVS mappings for lineage tracking. |
| **Version Discipline** | Stability | Commit atomic changes, tag releases, and document build order. |
| **Dependency Control** | Purity | Keep builds minimal, GF‑only, and document external integrations clearly. |
| **Documentation Upkeep** | Navigation | Maintain concise README files with installation steps and contribution guidelines. |

---

### 🔍 Maintenance Workflow

1. **Audit weekly** for schema drift, broken links, or dependency creep.  
2. **Update documentation** with short descriptions and topics for discoverability.  
3. **Run crawler checks** to confirm sitemap accuracy.  
4. **Apply patches** using structured artifacts only — batch scripts, AVIS headers, VB variants.  
5. **Lock builds** before major updates to prevent instability.  

---

### 🏗️ Architect’s Directive

Maintaining repositories is about preserving the **identity** of each project within the constellation. Enforce strict naming, metadata, and documentation rules so every repo remains stable, navigable, and ready for integration into larger systems.
