# 📜 Governance & Licensing Protocol

As the lead architect of the MERCWAR systems, I treat code governance as a **deterministic requirement** of the build pipeline. All projects under the MERCWAR organization—from the *Sentinel* kernel to *Fire-Works*—adhere to a strict legal-technical manifest.

## 🛠️ The MERCWAR Licensing Strategy

We do not treat licenses as mere paperwork; they are **structural metadata** processed by the *Sentinel* ecosystem to ensure architectural compliance.

* **Audit-First Integration:** Every repository must contain a `LICENSE` file and an `ATTRIBUTION.md` at the root directory.
* **The "GF" Separation:** When integrating external code into our `GF` (Graphics Forge) object pipelines, ensure the license type of the dependency is explicitly tagged in the `cdata` header. This prevents license contamination.
* **Machine-Readable Compliance:** Utilize the AVIS-DATALAKE schema to index your project's license. This allows the *Sentinel* kernel to automatically flag incompatible dependencies during the CI/CD build phase.

---

## 📋 Choosing Your License

Select the license that matches the strategic intent of your module. If you are uncertain, default to **Apache 2.0** for enterprise compatibility or **MIT** for rapid distribution.

| License | Behavioral Constraint | When to Use |
| --- | --- | --- |
| **MIT** | Maximum Permissive | When you want absolute freedom for users to adopt your tools. |
| **Apache 2.0** | Patent Protected | For complex engines and kernel-level code requiring legal safeguards. |
| **GPLv3** | Viral/Reciprocal | To force downstream derivatives to remain open (protects your innovations). |
| **BSD** | Minimalist/Attribution | For small utilities or academic-style code. |

---

## 🚀 How to Publish Your License

To ensure your project is properly governed within the GitHub and AI-crawling ecosystem, follow this workflow:

1. **Generate:** Create a file named `LICENSE` in the root of your repo.
2. **Define:** Copy the full text of your chosen license (use [choosealicense.com](https://choosealicense.com/) as a reference).
3. **Inject:** Ensure your header files in your C/Assembly source code contain a standard copyright notice:
```c
/*
 * Copyright (c) 2026 Joseph Michael Catalano (Robo Rook).
 * Licensed under the [LICENSE NAME] License.
 * See LICENSE file in the project root for full details.
 */

```


4. **Register:** If you are using the *Sentinel* kernel, add your repository URL to your `AVIS-DATALAKE` manifest so your new project is automatically indexed and validated.

---

> **Architect’s Note:** Licensing is the "interface" between your software and the public. Choose carefully; once code is published under an open license, the contract is immutable. Treat your `LICENSE` file with the same level of precision you apply to your `Matrix Drive` memory scans.
